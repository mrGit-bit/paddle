# Release Prep Workflow Version Bump Plan

## Context

- The workflow `.github/workflows/release-prep-no-ai.yml` updates both `CHANGELOG.md` and `paddle/config/__init__.py`, but the commit step currently stages only `CHANGELOG.md`.
- As a result, release PRs generated by GitHub Actions omit the expected `__version__` bump in `paddle/config/__init__.py`.
- Discovery files:
  - `.github/workflows/release-prep-no-ai.yml`
  - `paddle/config/__init__.py`
  - `specs/005-release-prep-version-bump.md`

## Spec Reference

- `specs/005-release-prep-version-bump.md`

## Objectives

- Ensure workflow release commit includes version bump in `paddle/config/__init__.py`.
- Preserve existing release branch and PR creation behavior.
- Keep failure behavior explicit when no release files are changed.

## Scope

### In

- Update staging/commit logic in `.github/workflows/release-prep-no-ai.yml` to include `paddle/config/__init__.py`.
- Validate YAML remains syntactically valid.
- Update `CHANGELOG.md` under `## [Unreleased]` with this behavior fix.

### Out

- No changes to version derivation logic.
- No changes to release branch naming or PR metadata structure.
- No changes to runtime code in `paddle/config/__init__.py`.

## Risks

- Risk: Overly broad staging could commit unintended files in CI workspace.
  - Mitigation: Stage only the two intended files explicitly.
- Risk: Commit step could still fail if expected files are unchanged due to no-op version/changelog run.
  - Mitigation: Keep staged-diff guard and fail fast with clear message.
